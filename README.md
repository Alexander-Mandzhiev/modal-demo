# Modal Demo

Демо-проект с кастомными UI-компонентами на React 19 без сторонних UI-библиотек.

## Запуск

```bash
npm install
npm run dev
```

## Структура

```
src/
├── components/         # Переиспользуемые UI-компоненты
│   ├── GenericModal/   # Модалка (портал, оверлей, Escape, клик вне)
│   ├── SingleSelect/   # Select с единичным выбором
│   ├── UniTextarea/    # Текстовое поле с поддержкой ошибок
│   ├── Button/         # Кнопка (primary / danger / ghost)
│   ├── Header/         # Шапка приложения
│   └── ProjectCard/    # Карточка проекта
├── pages/              # Страницы (бизнес-логика + композиция компонентов)
│   └── ProjectsPage/
├── constants/          # Справочные данные
│   └── timezones.js
├── App.jsx             # Layout-шелл
├── App.module.css
├── main.jsx            # Entry point
└── index.css           # Глобальные стили / reset
```

## Архитектурные решения

### CustomSelect — только single-выбор

`SingleSelect` поддерживает выбор **одного** значения из списка.
Поддержка множественного выбора (`isMulti`) намеренно **не добавлена** в этот компонент.

Причина: single-select и multi-select — это по факту два разных компонента
с точки зрения UX и внутренней реализации:

| | Single | Multi |
|---|---|---|
| Тип `value` | `string` | `string[]` |
| Поведение клика | Выбирает и закрывает дропдаун | Тоглит элемент, дропдаун остаётся открытым |
| Отображение выбранного | Одна строка текста | Чипсы/теги с кнопками удаления |
| Клавиатурная навигация | Enter выбирает и закрывает | Enter тоглит, Escape закрывает |
| Дополнительная логика | — | «Выбрать все», «Очистить», счётчик |

Попытка объединить оба режима через флаг `isMulti` приводит к тому, что компонент
обрастает ветвлениями `if (isMulti)` в каждой функции, становится сложно читаемым
и трудно поддерживаемым.

Рекомендуемый подход при необходимости multi-select:

```
components/
├── SingleSelect/       ← single (текущий)
├── MultiSelect/        ← multi (отдельный компонент)
└── _shared/
    └── SelectDropdown/ ← общий дропдаун-список, переиспользуется обоими
```

Общие части (рендер списка опций, позиционирование дропдауна, клик вне)
выносятся в `_shared/SelectDropdown`, а каждый компонент отвечает
только за свою логику выбора и отображения.

### GenericModal — портал поверх всего

Модалка рендерится через `createPortal` в `document.body` с `z-index: 2147483647`,
что гарантирует отображение поверх любых элементов, включая `popover` и `<dialog>`.

### Нулевые зависимости от UI-библиотек

Все компоненты написаны с нуля на `useState`, `useRef`, `useEffect`, `useCallback`
и `createPortal`. Единственные зависимости проекта — `react` и `react-dom`.
